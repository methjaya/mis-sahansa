WEEK 1

Introduction to Management Information Systems

What is MIS?

MIS are organisational systems that leverage technology, human resources, and processes to systematically collect, process, store, and distribute information to support decision-making and control.

Key DefinitionMIS = Technology + Human Resources + Processes → Collect → Process → Store → Distribute Information

Historical Evolution of MIS

EraDevelopment
1940s-1950sEarly data processing using punch card systems
1960sMainframe computers revolutionise data management
1970sIntroduction of Database Management Systems (DBMS)
1980sPersonal Computers, client-server architecture
1990sERP systems (SAP), relational databases, Internet impact
2000sCloud computing, mobile technologies, e-commerce rise
PresentAI, Big Data, IoT, real-time analytics

Role of MIS in Organisations

  • Streamlining operations - Automation of routine tasks
  • Decision support systems - Relevant information for effective decision-making
  • Strategic planning - Aligning technology with organisational goals
  • Data security and integrity - Ensuring confidentiality and reliability

Case Studies

Case 1: Enron Scandal - Ethical Failures in MIS

  • Enron collapsed in 2001 due to accounting fraud and manipulation of MIS data
  • Executives misrepresented financial health, hiding debt and inflating profits
  • Lesson: Robust ethical frameworks are essential; MIS must not be misused for fraud

Case 2: Amazon - Operational Efficiency

  • Uses MIS for supply chain: inventory optimisation, cost reduction, delivery speed
  • MIS algorithms power personalised recommendations
  • Automation and robotics guided by MIS enhance order fulfilment

Case 3: Healthcare - Electronic Health Records (EHR)

  • MIS enables management of EHRs, improving accessibility and accuracy
  • Decision Support Systems (DSS) aid healthcare professionals
  • Challenges: data security, interoperability across healthcare systems

Systems Theory & Organisational Systems

System Components

ComponentDescription
INPUTSResources, data, or materials that enter the system
TRANSFORMATIONOperations that convert inputs to outputs
OUTPUTSResults or products of the transformation process

Open vs. Closed Systems

Closed SystemOpen System
Rigid, impenetrable boundariesPermeable boundaries
Does NOT exchange info with environmentExchanges information, energy, or material with environment
Mechanical systems may be closedBiological and social systems are inherently open
Exam TipOrganisations are OPEN systems - they interact with their environment (suppliers, customers, government, etc.)

Feedback & Loops

  • Open Loop: No feedback mechanism; output does not influence input
  • Closed Loop: Has feedback; output is fed back as input to adjust the system
  • Feedback: Information about outputs/processes fed back as input to maintain steady state

Introduction to Industry 4.0 Technologies

What is Industry 4.0?

The fourth phase of the industrial revolution, originating in Germany in 2011. A digital manufacturing method powered by advanced, interconnected technologies that adapt to changes to avoid failures.

The Nine Pillars of Industry 4.0

TechnologyKey Role
Cyber-Physical Systems (CPS)Integrate processes; enable instant control, agility, efficiency
Internet of Things (IoT)Connects devices/sensors; real-time tracking
Big Data & Analytics (BDA)Analyse large datasets for insights, agility, sustainability
Cloud ComputingVirtual storage/apps; real-time access; supply chain integration
Autonomous RobotsReplace human labour; flexible, cooperative, independent
Additive Manufacturing (3D Printing)Layer-by-layer production; reduces complexity and lead times
Augmented/Virtual Reality (AR/VR)Support warehousing, training, product design
SimulationsRecreate real-world data virtually; optimise before physical changes
Cyber SecurityProtect data integrity and connectivity from cyber threats

Industry 4.0 in Sri Lanka's Apparel Industry (Sachin & Bandara, 2021)

Technologies Adopted

AR/VR Big Data & Analytics Cloud Computing IoT Robotics

Barriers to Adoption

  • Financial: High initial investment costs
  • Legal: Regulatory barriers (e.g., RFID frequencies not approved)
  • Cultural: Reluctance to store data on cloud
  • Short-term focus by decision makers

Impact on Performance

  • Organisational: Improved efficiency, effectiveness, competitiveness
  • Supply Chain: Cost reduction, lead time reduction, quality improvement, information sharing
  • Key technologies: Big Data, Cloud Computing, IoT

Model Evaluation Metrics

Classification Metrics

Accuracy = (TP + TN) / (TP + TN + FP + FN) Precision = TP / (TP + FP) Recall = TP / (TP + FN) F1-Score = 2 × (Precision × Recall) / (Precision + Recall) ROC-AUC → ideal = 1.0, random = 0.5

Regression Metrics

  • MSE - Mean Squared Error; penalises large errors heavily
  • RMSE - Root MSE; error in same units as target; easier to interpret
  • R² - Explains variance (0 to 1; negative if worse than mean)

Confusion Matrix

Predicted: YESPredicted: NO
Actual: YESTP (True Positive)FN (False Negative)
Actual: NOFP (False Positive)TN (True Negative)

WEEK 12

Introduction to Database Management Systems (DBMS)

Key Definitions

  • Data: Known facts that can be recorded and have an implicit meaning
  • Database: A highly organised collection of assembled data
  • Mini-world: Part of the real world about which data is stored
  • DBMS: Software package to facilitate creation and maintenance of a computerised database

Main Characteristics of the Database Approach

  • Self-describing nature: DBMS catalog stores metadata
  • Program-data independence: Data structures can change without changing DBMS access programs
  • Data abstraction: Data model hides storage details
  • Multiple views: Each user sees only data of interest
  • Multi-user transaction processing: Concurrency control; OLTP

Three-Schema Architecture

Schema LevelDescriptionManaged By
External Schema (User View)Individual user/application views; hides complexityApplication programs
Conceptual Schema (Logical View)Overall structure: entities, attributes, relationships - blueprintDatabase designers
Internal Schema (Physical View)Physical storage: file organisation, indexing, data typesDatabase administrators

Data Independence

  • Logical Data Independence: Change conceptual schema without affecting external schemas or applications
  • Physical Data Independence: Change internal schema without changing the conceptual schema

ACID Properties (Transaction Processing)

PropertyMeaning
AtomicityTransaction is all-or-nothing - indivisible unit of work
ConsistencyDatabase remains in valid state before and after transaction
IsolationConcurrent transactions do not interfere with each other
DurabilityOnce committed, effects are permanent - survive system failures

DBMS Languages

LanguagePurpose
DDL (Data Definition Language)Define database structure, data types, constraints
DML (Data Manipulation Language)Retrieve and update data
SQL (Structured Query Language)International standard; set-oriented; declarative

Client-Server Architectures

ArchitectureDescription

Three-TierClient + Application/Web server (business logic) + Database server; enhanced security

WEEK 14

Security & Ethical Challenges of IT and E-Business

Key Definitions

  • Cyber Security: Protection of information systems, data, and services from unauthorised access, harm, or misuse
  • Cybercrime: Use of a computer as an instrument to further illegal ends

Social Engineering Attacks

Steps of an SE Attack

  • 1. Preparation - Gather info about victims via social media, email
  • 2. Infiltration - Approach victim impersonating a trustworthy source
  • 3. Exploitation - Use persuasion to request account logins, payment info
  • 4. Disengagement - Stop communication, commit attack, and depart

Human Tendencies Exploited

Authority Liking Reciprocation Consistency Social Validation Scarcity

Insider Threats

Malicious InsidersPurposely misuse access to steal data or sabotage systems
Negligent InsidersNo malicious intent; compromise security by falling for phishing
Compromised InsidersExternal actors coerce or bribe them to provide access

Security Best Practices

  • Network Security: Firewalls, IDPS, network segmentation, regular patching
  • Access Controls: Strong authentication (MFA, biometrics); principle of least privilege
  • Encryption: Data at rest and in transit; SSL/TLS
  • Secure Coding: OWASP Top Ten; code reviews; penetration testing
  • Incident Response: Documented plan; drills; post-incident analysis
  • Security Awareness Training: Regular training on phishing and social engineering

Ethical Considerations in E-Business

  • Privacy and Data Usage - Transparent policies; informed consent
  • Responsible Data Collection - Collect only necessary data; handle securely
  • Ethical Online Marketing - No deceptive advertising
  • User Profiling - Inform users; provide opt-out options
  • E-Business Fraud - ML detection systems for payment fraud, account hijacking
